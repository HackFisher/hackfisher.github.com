假设一个国家中只有两种人，这两种人分别具备两种能力。

思维轨迹。

或者，假设一个人只有两种能力，"Big Picture"的能力和把问题细分的能力。解决最大的问题是上界resovle(MAX)，解决最小的问题是下界resolve(MIN)。

Int x; MAX MIN;

getX();

A具有所有的x, resovle的范围受限

resolve(x) {

Return x;

}

B具有ResovleA，getX()的范围受限

那个更可能取得成就。。。违反直观的是，两者是平等的，不确定，取决与他们的变量能力。

也就是，有全局观的人，和有能力做事的人同样有机会。

这些仅仅是理论，事实更可能是A可以去resolve(MAX)，却并不一定有resovle（MIN），因为resovle的方法是可以举一反三的，resovle(MAX)和resovle(MIN)并没有不同。resolve往往取决于聪明抽象能力

但B却往往取不到MAX，因为get(MAX)和get(MIN)相差甚远。Get()往往取决于得到资源的能力。

所以事实情况是，A往往决策，而B具体做事，但不绝对意味着A能比B做的好。

能看清楚"Big Picture"的能力和把问题细分的能力，分别对应于一个人所能取得成就的上界和下界。

分析：看清楚"Big Picture"的能力的人更能够抽象归纳问题，也就是说，能够提出更广泛，更具挑战性和普遍使用性的问题，问题越大，越普遍，也就更能发掘更有价值的东西，能够统一，从而更具有创新，能够涵盖更广泛的领域，能够解决更大的问题。如果不具有这种能力，那么连最基本的问题目标都没有提出来，更不要谈去解决它了。所以，仅仅是上届，基本面，意味着可能性，解决的问题必须是提出的问题的一部分，但是提出问题不一定会被解决。

提出归纳问题后，还要解决它，这就需要所说的第二种能力，问题细分为更小问题的能力，具有这种能力的人只意味着：给他一个问题，让他去解决，他具有能力去解决这个问题，但不意味着他又能力提出这个问题。也就是说如果他没有提出更大问题的能力，他甚至连解决更大问题(理想化，当然可能别人提出问题，他来解决，这就是合作两种能力互补的力量)的机会都没有。所以他只能解决那些他能提出来的较小的能力，他能确保解决他，也就是说保证了一个下界，这个下界就是他能提出来的最大问题，但是因为他的能力很低，所以相对于前一种能力来说，是他后一种问题的下界，因为他本具有能力解决更大的问题。
